# Security Policy

## Reporting a Vulnerability

**Please do not report suspected vulnerabilities via public GitHub issues.**

Preferred channel:

- [Report a vulnerability privately via GitHub](https://github.com/culmat/information-classification/security/advisories/new)

Fallback:

- Email <culm@culm.at>

Please include steps to reproduce, affected versions, and any proof-of-concept material.

## Response SLA

- Response within **5 business days**
- Business hours: Mon–Fri, 09:00–17:00 CET

## Scope

In scope:

- The Forge app source in this repository (`information-classification`)

Out of scope:

- The Atlassian Forge platform and Confluence itself — report those to [Atlassian](https://www.atlassian.com/trust/security/vulnerability-disclosure-program)
- Third-party services — the app makes no outbound network calls and has no external dependencies at runtime

## Supported Versions

Only the version currently published on the [Atlassian Marketplace](https://marketplace.atlassian.com/) is supported. Fixes are not back-ported to older versions.

## More Information

The full security, privacy, and compliance posture is documented in the [Trust Center](https://culm.at/information-classification/trust-center/#security).
